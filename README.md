# 九州多账号状态管理

> 一个面向《九州》的多账号状态面板，适合同时管理多个账号的登录、状态查看与常用自动化操作。  
> 提供 **Tampermonkey 脚本版** 和 **独立 HTML 页面版**。

## 这个工具能做什么

### 账号与登录
- 多账号管理
- 单个新增 / 批量导入账号
- 用户名 + 密码登录
- 自动识别验证码模式：
  - 本地图片验证码
  - 腾讯点击验证码
- 支持手动清空本地 token

### 状态查看
- 角色信息：昵称、称号、境界
- 体力与回满倒计时
- 功法自研冷却
- 伙伴招募冷却
- 挂机状态、累计收益、战斗统计
- 灵石 / 银两统计
- 稀有物品统计：
  - 修行月卡
  - 顿悟符
  - 高级招募令
  - 易名符
- 全局汇总卡片（总灵石 / 银两、总稀有物品、上次自动刷新）

### 自动化能力
- 自动刷新全部账号状态（分钟级，可关闭）
- 自动挂机：复用服务端最近一次挂机配置
- 自动秘境：
  - 读取秘境列表
  - 指定秘境 ID 与难度
  - 自动创建实例、启动战斗、跟踪会话、推进流程
  - 自动读取战斗技能；读取失败时以普攻兜底
- 宗门功法残页兑换：
  - 单账号兑换 500 残页
  - 全局批量兑换 500 残页
  - 贡献不足时自动捐献灵石后重试
  - 今日额度不足 500 时按剩余额度兑换
- 浏览器通知提醒：功法自研 / 伙伴招募冷却完成时弹出通知
- 多种排序：手动、名称、功法剩余、招募剩余、体力高到低、体力低到高

## 文件说明

- [`jiuzhou-multi-account-status.user.js`](./jiuzhou-multi-account-status.user.js)：Tampermonkey 脚本版，嵌入游戏页面侧边面板
- [`index.html`](./index.html)：独立页面版，默认全屏双栏布局（左侧账号列表，右侧账号详情）

## 快速开始

### 方案一：Tampermonkey（推荐）

1. 安装 [Tampermonkey](https://www.tampermonkey.net/)
2. 安装脚本：
   - [Raw 安装链接](https://raw.githubusercontent.com/yuai404/jiuzhou-multi-account-status/main/jiuzhou-multi-account-status.user.js)
   - 或手动打开仓库内的 [`jiuzhou-multi-account-status.user.js`](./jiuzhou-multi-account-status.user.js)
3. 打开 `https://jz.faith.wang/`
4. 点击右下角 **“打开多账号”**
5. 添加账号，登录后点击 **“刷新状态”**

> 当前脚本 `@match` 仅覆盖：
>
> - `https://jz.faith.wang/*`
> - `http://localhost:*/*`

### 方案二：独立 HTML 页面

1. 打开 [`index.html`](./index.html)
2. 页面默认使用 `https://jz.faith.wang/api`
3. 如需对接兼容后端，可在面板顶部修改 `API Base`
4. 如果浏览器对本地 HTML 的远程请求、CORS / CSP 或腾讯验证码 SDK 有限制，请改用 Tampermonkey 方案

## 使用说明

### 1. 添加账号 / 批量导入

支持以下任一格式，一行一个账号：

```text
备注,用户名,密码
用户名,密码
```

也支持：

- 制表符分隔
- `|` 分隔

说明：

- 密码只进入当前页面内存
- 不会写入 `localStorage`

### 2. 登录与验证码

工具会先读取：

- `/captcha/config`

然后自动切换登录流程：

- `local`：显示图片验证码输入框，点击 **“刷新验证码”** 后走 `/auth/captcha`
- `tencent`：点击 **“登录”** 后自动弹出腾讯点击验证码

如果后端切换了验证码方案，可点击面板顶部的 **“刷新模式”**。

### 3. 刷新状态

点击 **“刷新状态”** 后，会读取账号的核心状态，包括：

- 角色信息
- 体力与恢复进度
- 功法自研冷却
- 伙伴招募冷却
- 挂机状态
- 灵石 / 银两
- 稀有物品

点击 **“刷新全部状态”** 可批量刷新所有已登录账号。

### 4. 自动挂机

点击 **“开始挂机”** 时，工具会优先读取服务端的最近一次挂机配置：

- `/idle/config`

因此你需要先在游戏内手动配置过至少一次挂机参数。

工具在全局自动刷新后，如果检测到：

- 账号已登录
- 已有可用挂机配置
- 当前不在挂机
- 当前没有冲突战斗会话

则会尝试自动续挂。

### 5. 自动秘境

使用前：

1. 在账号卡片中填写 **秘境 ID**
2. 选择或输入 **难度**
3. 点击 **“开始自动秘境”**

工具会自动：

- 从 `/dungeon/list` 读取秘境候选列表
- 创建秘境实例
- 启动战斗会话
- 跟踪当前 battle session
- 自动推进秘境流程
- 记录自动秘境日志

说明：

- 如果战斗技能配置读取失败，会以普攻兜底
- 如果当前已存在冲突战斗会话，自动秘境会停止并提示原因
- 点击 **“停止自动秘境”** 后，不会再继续新的秘境流程；已开启的当前战斗会自然结束

### 6. 宗门功法残页兑换

支持：

- 单账号 **“兑换 500 残页”**
- 全局 **“全局兑换 500 残页”**

执行时会检查：

- 是否已加入宗门
- 宗门商店中是否存在功法残页商品
- 当前贡献是否足够

特殊处理：

- 如果今日剩余兑换额度不足 500，会自动按剩余额度兑换
- 如果贡献不足，工具会尝试自动捐献 `2500` 灵石后重试

### 7. 浏览器通知提醒

使用步骤：

1. 勾选 **“启用浏览器提醒”**
2. 点击 **“请求授权”**
3. 保持页面或脚本处于运行状态

当以下冷却结束时会弹出浏览器通知：

- 功法自研
- 伙伴招募

## 配置项说明

| 配置项 | 说明 |
| --- | --- |
| API Base | 默认是 `https://jz.faith.wang/api`，可改为兼容后端地址 |
| 自动刷新 | 单位为分钟，`0` 表示关闭 |
| 排序方式 | 支持手动、名称、功法剩余、招募剩余、体力排序 |
| 冷却完成提醒 | 控制是否启用浏览器通知 |
| 验证码模式 | 读取 `/captcha/config` 后显示当前模式，可手动刷新 |

## 数据存储与安全

会写入 `localStorage` 的内容包括：

- `apiBase`
- 自动刷新设置
- 通知设置
- 排序方式
- 账号列表
- token
- 最近一次读取到的角色 / 挂机 / 冷却 / 物品等状态

不会写入 `localStorage` 的内容：

- 登录密码
- 图片验证码输入内容
- 腾讯验证码临时参数

补充说明：

- 密码只保存在当前页面内存
- 如果同一账号在别处登录，旧 token 可能失效，需要重新登录

## 已知限制 / 常见问题

### 1. 为什么独立 HTML 版打不开验证码或请求失败？
独立 HTML 版可能受到浏览器对本地文件、跨域请求、CSP 或腾讯验证码 SDK 加载策略的限制。遇到这类问题时，优先使用 Tampermonkey 版。

### 2. 为什么自动挂机不能直接开始？
自动挂机依赖 `/idle/config` 返回最近一次挂机配置。如果你从未在游戏内手动配置过挂机参数，工具无法凭空生成配置。

### 3. 为什么自动秘境会中断？
常见原因：

- 体力不足
- 当前已有其他战斗会话
- 秘境 ID 不正确
- 服务端战斗状态长时间未推进

### 4. 为什么通知没有弹出？
请检查：

- 浏览器是否已授权通知
- 面板中是否勾选了“启用浏览器提醒”
- 页面 / 脚本是否仍在运行

### 5. 为什么 README 里没有本地绝对路径？
这是一个面向 GitHub 在线阅读的 README，文档中统一使用仓库相对路径和在线链接，避免依赖作者本机目录结构。
